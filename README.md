![Status](https://img.shields.io/badge/status-operational-green) ![Version](https://img.shields.io/badge/version-1.0.0-blue)

## 🎯 Цель проекта

Разработать аналитического AI-агента, способного на основе данных API OWS v3 формировать проверяемые ответы, выявлять аномалии и оценивать эффективность закупочных процессов.

## 🚀 Быстрый старт

### Требования
- Python 3.12+
- PostgreSQL 15+
- Docker & Docker Compose

### Установка

```bash
# 1. Клонировать репозиторий
git clone <repo>
cd goszakup_ai_agent

# 2. Создать виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать .env файл
cp .env.example .env
# Заполнить токены API и параметры БД
```

### Запуск через Docker Compose

```bash
docker-compose up -d
```

Это запустит:
- PostgreSQL контейнер с инициализацией БД
- Приложение (синхронизация данных)

### Синхронизация данных

```bash
# Полная синхронизация всех 27 организаций
python3 sync_all.py

# Или отдельно:
python3 sync_contracts.py     # Контракты
python3 sync_announcements.py # Объявления
python3 sync_lots.py          # Лоты
python3 sync_subjects.py      # Субъекты
```

### Запуск AI-агента

```bash
# Интерактивный режим с вводом вопросов
python3 ai_agent.py

# Пример вопроса:
# "Найди закупки с отклонением цены > 30% по ЕНСТРУ 986652"
# "Покажи лоты для ЕНСТРУ 0"
# "Оцени справедливость цены 500000 тенге для ЕНСТРУ 986652"
```

## 🏗️ Архитектура

```
API OWS v3 (GraphQL)
        ↓
    [ETL Пайплайн]
        ↓
    PostgreSQL DB
        ↓
   [Analytics Layer]
        ↓
    [AI Agent]
        ↓
    LLM (Nitec AI)
        ↓
   Structured Response
```

### Компоненты

1. **Data Ingestion** (`sync_*.py`)
   - GraphQL запросы к OWS v3 API
   - Инкрементальное обновление (< 24h delay)
   - Обработка ошибок и retry логика

2. **Database** (`database/schema.sql`)
   - 11 таблиц с нормализованной схемой
   - JSONB для гибкого хранения данных
   - Индексы для оптимизации

3. **Analytics** (`analytics.py`)
   - Статистические методы (медиана, IQR, персентили)
   - Обнаружение аномалий (Tukey's fences)
   - Регрессионный анализ трендов

4. **AI Agent** (`ai_agent.py`)
   - Классификация типов вопросов
   - Извлечение параметров (ЕНСТРУ, БИН, суммы, даты)
   - Обращение к LLM с контекстом
   - Структурированный вывод

## 📋 Функциональность AI-агента

### Типы вопросов

#### 1️⃣ Поиск
```
Вопрос: "Покажи лоты для ЕНСТРУ 986652"
Ответ: Список найденных лотов с суммами и описаниями
```

#### 2️⃣ Аномалии
```
Вопрос: "Найди закупки с отклонением цены > 30% по ЕНСТРУ 0"
Ответ: Количество аномалий, их характеристики, примеры
```

#### 3️⃣ Справедливая цена
```
Вопрос: "Оцени справедливость цены 500000 тенге для ЕНСТРУ 986652"
Ответ: Сравнение с рыночной ценой, оценка адекватности
```

#### 4️⃣ Объемы
```
Вопрос: "Выяви нетипичное завышение количества в 2024 году"
Ответ: Анализ тренда, отклонения, прогнозы
```

## 🔄 Инкрементальное обновление

Система поддерживает автоматическое обновление с задержкой < 24 часов:

```python
# sync_meta таблица отслеживает последнее обновление
SELECT * FROM sync_meta;

entity        | last_update_date
--------------+-----------------------
announcements | 2026-04-26 11:07:51+00
contracts     | 2026-04-26 11:06:45+00
lots          | 2026-04-26 11:05:30+00
```

При следующем запуске `sync_all.py` загружаются только новые/обновленные записи.
